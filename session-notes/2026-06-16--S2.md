## 4. 🌟 `SwapSet` — the first genuinely-new card (Double "swap a set, both purged")
`PropertyActionKind.SwapSet` + `PropertyActionService.SwapSet`: holder picks one of their complete
buildable sets, a target player holding a complete set, and which of the target's sets to take;
every title in each set is exchanged (`Transfer`), then **both** swapped sets are purged (John had

The keystone for the whole Tax deck:

  receipt/toast, not a generic card charge.

## 6. 🌟 `DiceOffPlayer` target (Tax "lowest pays" / "lowest pays + swaps with you")
New `PlayerTarget.DiceOffPlayer` — the dice-off winner is the **subject** (pays to Bank/FP), not a
player-to-player counterparty. `MoneyActionService.ApplyToDiceOffSubject` resolves the dice-off and
**stashes the winner on `CardActionContext.DiceOffPlayerId`**, so a later action in the same group
(card 444's `Movement{Swap, Target=DiceOffPlayer}`) acts on the same player (`ResolveCard` passes one
context instance to every action). Set `IncludeHolder=false` on both (only others roll; the holder
escapes their tax, and the swap is guaranteed to be someone else).

## 8. 🌟 Engine changes the JSON forced (with verification)
- **`CardTrigger` +`OnTaxLanded` (1<<15) +`OnSnakeEyes` (1<<16)** — card 21 needed a tax trigger the
  enum lacked; card 24's "snake eyes" can't be a refinement of `OnRollDouble`, so it got its own flag.
- **`NoOpAction` + `NoOpActionService`** (+ DI + `[JsonDerivedType("NoOp")]` + `CardService` arm) —
  for card 31, a **suppress-only** card whose entire effect is `SuppressDefault{SuppressFreeParkingMoneyTake}`;
  the NoOp gives the group its required ≥1 action.
- **🔴 `SuppressDefault` was undeserialisable.** Its only ctor takes `SuppressDefaultType`, which
  binds to no property → System.Text.Json throws `ConstructorParameterIncompleteBinding` for **any**
  card carrying a `SuppressDefault`. Existing decks only worked because they all omit it. **Verified
  via a throwaway xUnit test** (failed 3/3), fixed by adding a **parameterless constructor** (so the
  bool flags deserialise directly, e.g. `{"SuppressGoBonus":true}`), re-verified green, then a second
  temp test deserialised the whole `third.json` into the model (all polymorphic `$type`s, both suppress
  flags, `TurnsActive`, `DiceOff`, `SnakeEyesBonus`, `AmountSource`, the new triggers) — green; temp
  tests deleted. **This bug would have broken every Tax card, both global-suppress Doubles, and the
  four triple-bonus Triples on import.**
