## 4. Reservation-rule deactivation

John added `GameModel.CheckReservationRule(playerId)` for the **deadlock** path
(everyone else already reserved → the last player completing a set breaks it). On
review I caught + fixed a **regression** the refactor introduced:
`ProcessUnownedProperty` was routing to `ReserveProperty` on `ReserveRuleActive`
*alone*, dropping the `buyingLastInSet` half of the old gate — so any unowned
landing during the reserve phase wrongly offered a half-price reservation. Fixed
to `buyingLastInSet && ReserveRuleActive`.

Then implemented **#3** (the missing path John flagged): an auction winner
completing a set ends the mechanic. New
`GameModel.CheckReservationRuleSetObtained(playerId)` — disables the rule when the
player holds a complete colour set (excludes reserved; stations/utilities never
trigger). Called after the win in `UnownedProperty`. Updated `auction-flow.md` §7.

The four ways the rule turns off (John's spec): (1) a deal gives someone a set,
(2) Free Parking, (3) an auction win, (4) the deadlock. #4 = `CheckReservationRule`;
#3 wired; #1/#2 await Deal/FreeParking services (same `CheckReservationRuleSetObtained`
hook). Flagged that `game-rules.md` Reserved Properties **rule 6** ("ends once
*every* player has a set") doesn't match the implemented "any player breaks
through / deadlock" model — left for John as rules author.

## 5. `game-rules.md` rewrites

  only** (Free Parking dropped); instalment is **10% of the total originally
  borrowed across all currently-outstanding loans**, rounded per the game rule

## 8. Architecture — where player commands live

Decided **domain-service ownership**, not a `PlayerService` façade: portfolio
actions live on `PropertyService` (the `Try*` methods), jail on `JailService`,
bankruptcy → a future `BankruptcyService`, etc. The commands are single-domain
(no cross-service composition), the capability gate already lives on
`TurnStateProvider` (re-checked on the writer thread), and the web
`PlayerProfileService` is the documented command surface. A façade would only
duplicate the gate or become a coupling hub.

## 11. 🌟 Major: prompts as server-rendered profile state (Approach A)

The big one. Reframed engine prompts from client-toggled Bootstrap modals to
**server-rendered state inside `_PlayerProfileView`**. Done in three chunks:

- **Chunk 1 (render):** `_PlayerProfileView` reads `cache.PendingPrompt`, and if
  it's for this player renders a `.pp-prompt` overlay (`data-prompt-id` /
  `-type` / `-stamp`) switching on type → the matching fragment. The four prompt
  partials rewritten from hidden modals into typed `@model` fragments with submit
  hooks (`data-prompt-submit` / `-decline` / `data-bid-amount`); colour-by-set and
  the `Type`-driven wording moved server-side. Page-level prompt includes removed
  from both pages.
- **Chunk 2 (submit + drawer):** new `ingame-prompts.js` — one delegated submit
  module that reads the rendered `.pp-prompt` and calls `SubmitPrompt`.
  `player-drawer.js` rewritten: **auto-open on the prompted player, no auto-close,
  no prompt-lock** (close / prev / next / Esc / backdrop always work). Deleted the
  four old per-type handlers; swapped the page `<script>`s.
- **Chunk 3 (CSS):** `.pp-prompt` overlay scoped to the drawer (absolute → the
  transformed `.player-drawer` is its containing block, so it dims only the
  drawer, never the board) and to the viewport on the phone; z-order tabbar 1030 <
  overlay 1031 < drawer header 1032 (so close/switch stay usable mid-prompt).
  Drawer width ×1.6 (380 → 608px).

What it delivers (confirmed with John): a prompt **cannot be closed, only
answered** (every control submits a response); it stays rendered while pending and
is SignalR-synced across the phone and host renders; the host can close the drawer
/ switch players / use the board without answering, and the prompt reappears when
they return; and the **multi-prompt race is gone** (no client open/close to
mis-order — the prompt is just swapped state). The interruptible/NOPE window
(game-view, host-pause) is explicitly out of scope and unbuilt.
