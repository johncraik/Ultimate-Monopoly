## 4. 🌟 Pagination route-key collision (bug)

`PageNumber` was stuck on **1**. Cause: **`page` is a reserved Razor Pages route key** (holds the page
path), and the **route** value provider wins over the query string, so `[BindProperty(Name="page")]`
bound `"/Users/Index"` (unparseable → default). Fix: bind/emit **`pageNumber`** instead (page model +
`href-format` + the JS). Commented so it isn't "corrected" back.

## 7. 🌟 Disabled Account page (`/Identity/Account/Disabled`)

Pairs with the admin Disable action. **`Program.cs`:** `AddIdentity(configureMiddleware: o =>
o.AccessDeniedRoute = "/Identity/Account/Disabled")` — the identity middleware redirects `IsEnabled ==
false` users here, and `ExcludedPaths` is auto-built from `AccessDeniedRoute`, so a disabled-but-signed-in
user can actually load it. Generic 403 still → the existing `AccessDenied` (separate cookie path).
Page model holds all logic (no service): **Log out** + **self-Delete** (with confirm modal stressing
irreversibility) → sign out → home.
