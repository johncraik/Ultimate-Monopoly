## 2. 🌟 Card-system runtime fixes

- **Any-player trigger fired on its own holder** (`CardTriggerService.FilterToContext`). The "former prisoner steals the GO bonus" (`OnOtherPassGo`, `MetAnyPlayerTurn`) fired on the passer when the passer *was* the holder, because `holder == subject` returned `true` for any condition type. Fix: when the holder **is** the subject, only `*CardholderTurn` cards apply; any-player cards require the holder to be a bystander. Centrally fixes the FP-take and triple-cancel siblings too.
- **"After your next move" card auto-fired on the move that drew it.** JV "forward 23 / back 17" (`OnNextMove`) is drawn during `ResolveBoardSpaceForPlayer`, and the orchestrator's `OnNextMove` fires on the next line. Fix: `CardModel.DrawnOnTurn` stamped in `DrawCard`; `MatchingCardForTrigger` gates **only `OnNextMove`** on `Metadata.TurnNumber > DrawnOnTurn`. Recursion-safe (turn number constant through the evaluation); covers both own-roll and third-die-move cases.
- **Swap-with-jailed-player didn't release the lock** (`MovementActionService.ApplySwap`). A swap pulls a jailed target out, but their forced-stay flags persisted. Fix: capture `target.IsInJail` *before* the index swap, then `_jailService.ResetPlayerJailFlags(target)` (made **public**) after.
- **Mass breakout left stale jail flags** (`MovementActionService` `GoToJustVisiting`). It's a `Movement` (`Everyone`/`OnlyJailed`), not a `Release` — a bare teleport to Just Visiting that bypassed `JailService`, so `MinJailTurns`/`JailTurnCounter` lingered and re-jailed the player. Fix: jailed targets route through `JailService.LeaveJailByCard` (resets flags + advances), non-jailed keep `AdvancePlayer` honouring `CollectGoBonus`. (Re-modelling as `JailKind.Release` would only free the first target — the `JailActionService` Release case `return`s inside the loop.)
- **Leaving jail by paying prompted for the GOOJF card.** `LeaveJailByPaying` fires `OnInJail`, which surfaced get-out-of-jail-free (`ChoiceCardholderTurn` + `Trigger=OnInJail`) as a Choice prompt even though the player chose to pay — GOOJF is handled independently (`LeaveJailCard` hub command / `LeaveJailPrompt`). Fix (John's call, defensive): set the GOOJF cards' `Trigger` to **`0` (None)** in all 4 decks (chance/comChest/percentChance/percentComChest) but **keep** the `Conditions` array + `ConditionType`. `None.HasFlag(OnInJail)` is false → never trigger-surfaced; befriend-a-guard (`ModifyLeaveFee`, still `OnInJail` 512, forced) keeps auto-waiving the fee. GOOJF identified by `JailAction{Kind=Release}` (`PlayerModel.GetOutOfJailCard()`).

## 3. 🌟 Jail / transactions / bankruptcy

- **Collect-rent-in-jail did nothing** (transaction layer, John fixed). `TransactionService.PayRent` re-checked the owner's jail state as a "safeguard" but omitted the `CollectRentInJail` exception that `PropertyService.PropertyRent` has — so rent was computed, the payer acknowledged it, then the guard `return`ed before the money moved. Fix: guard now `owner is { IsInJail: true, CollectRentInJail: false }` only.
- **Bankruptcy returns held cards to the deck** (John). Each card's groups are reset (`TurnsRemaining = TurnsActive`, `IsChosenGroup = false`) before `CardDecks.HandBack` — mirrors `PlayCard`'s spent path. Suggested (and John added) `player.Cards.Clear()` after the loop so the bankrupt hand doesn't keep references to deck-resident instances (the profile cards tab isn't gated on `IsBankrupt`).
- **>2-player bankruptcy crash** (John). `ProcessBankruptcy` now captures `currentPlayer` *before* `IsBankrupt = true` (since `CurrentPlayer()` returns null once flagged), so the "was the bankrupting player the current player?" guard is reliable — turn-state only mutates when the current player bankrupts; a non-current player (player C) bankrupting mid-someone-else's-turn no longer corrupts the turn.

## 4. 🌟 Card receipts finalised + a real overflow bug

`CardTakenReceipt` / `CardPlayedReceipt` finalised with `(CardModel, GameCacheModel, playerId)` constructors computing `AllTriggers`, `ConditionType`, `IsImmunity`, group/action counts, display text. Emit sites updated (`DrawCard` keep branch, `ResolveCard` after `IsChosenGroup` is set).

**Bug fixed:** the trigger aggregation cast each flag to `(ushort)`. `CardTrigger` is `int`-backed with flags above 16 bits (`OnSnakeEyes 1<<16`, `OnTurnStart 1<<17`, `OnSpaceLand 1<<18`) — those truncate to 0 under `ushort`, so anytime cards (`OnTurnStart|OnSpaceLand`) logged `AllTriggers = None`. Changed to `(int)` in both receipts. (The `try/catch` around it is unreachable — `Conditions` never null, `Trigger` a value type — noted but left.)
