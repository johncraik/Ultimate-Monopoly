## 1. 🌟 SuppressDefault aliasing (§9 from 2026-06-17) — fixed, and the trigger now returns it
`CardTriggerService.EvaluatePlayableCards` no longer aliases card #1's `SuppressDefault` definition as
the accumulator: it builds a **fresh** one (`new SuppressDefault(suppressDefaults.Type())`, via a new
`SuppressDefault.Type()` round-trip) and aggregates into that. Public trigger methods now return
`Task<SuppressDefault>` (the aggregated suppress), and `ProcessGameTrigger` returns it. Class doc
refreshed (no longer "returns Task for now").

## 2. 🌟 GoService — the first wired call site (and three follow-on fixes)
`OnLandGo` wired into `LandOnGo`; `OnPassGo` + `OnOtherPassGo` into `CollectGoMoney`. Fixed in review:
- **`OnOtherPassGo` result was discarded** → now gates the passer's bonus (a bystander "former
  prisoner" steal suppresses it). The other pass-GO consequences (mortgage fee, loan repay,
  `HasPassedInitialGo`) run regardless — only `ReceiveGoBonus` is gated (no early-return).
- **`LandOnGo` didn't fire `OnLandGo`** at all (held GO cards never fired) → now does, combined with
  the drawn-card suppress.
- **`LandOnGo` mutated the drawn card's definition** (`suppressDefault.Aggregate(triggerSuppress)` on
  the `DrawCard` return = the §9 footgun again) → now reads a fresh `sd`; the check reads `sd`, not the
  draw-only `suppressDefault` (that bug silently leaked the default £200 on top of the doubled card).

## 3. 🌟 `CardOptionPrompt` — two render modes
The partial (`_CardOptionPrompt.cshtml`) now branches on `PlayCardChoice`: **group choice** (mandatory
radios + Confirm, unchanged) vs **play-card** (single → yes/no with a **Don't play** decline;
multi → radios + Don't play). `ingame-prompts.js` `CardOption` case honours `[data-prompt-decline]`
(empty key) and a single-card `data-card-option-key` on the submit button. The validator already
permits the empty key when `PlayCardChoice`. **Card selection (steal/pass) reuses the mandatory mode —
no new flag** (it's just a "pick one" over a hand).

## 4. 🌟 Anytime cards — `OnTurnStart` + `OnSpaceLand`, and dynamic occasions
John's call (option B): "anytime on your own turn" = two real windows. Added `CardTrigger.OnTurnStart`
(1<<17, fired by the play-a-card **command** in `PlayerProfileService`) + `OnSpaceLand` (1<<18, fired by
the engine in `BoardService.ResolveBoardSpaceForPlayer`). The 9 anytime keep-cards carry
`Trigger=OnTurnStart | OnSpaceLand` (393216) — **except "Change direction"**, which John set to
`OnTurnStart` only. `CardTrigger.None` reverts to "resolve-on-draw only". John also made the **occasions
count dynamic** (`{OC__<i>}` → group `TurnsRemaining`, decoupled from money actions in `FormatCardText`).

## 11. 🔴 OPEN BUGS — card runtime issues (John-reported, undiagnosed)

John hit a cluster of card bugs in play; recorded here for the next session. **Not yet diagnosed** —
these are suspect areas, not confirmed root causes.

- **SuppressDefault issues.** Suppress consumption is **uneven across call sites**. Known: the FP
  sub-action checks in `ProcessFreeParking` still read the **drawn-card** `suppressDefault` (fine /
  property-take / hand-in / purge at the path-A/C branches) rather than the combined `sd`; only the
  money-take was switched to `sd`. The **dice triggers' suppress** (cancel-triple-bonus's
  `SuppressTripleBonus`, the snake-eyes redirect) needs its result consumed where `RollTurnDice` returns
  the tuple — confirm the orchestrator honours it. Start by auditing every `ProcessGameTrigger` caller:
  does it read the returned `SuppressDefault`, and against `sd` (combined) not the draw-only value?
- **Chained-play issues.** `EvaluatePlayableCards` re-evaluates after each play (to let other cards
  react). Suspect the chaining: the `playedCards` guard (by `CardId`), multi-use re-add **order** (a
  re-added card lands at the end of `Cards`), forced-before-choice ordering, and re-entrancy when a
  played card's effect itself fires a trigger (e.g. an advance card's landing → `OnSpaceLand`). The
  self-retrigger loop is fixed (§9), but cross-card / nested chains may still misbehave.
- **Prompt target issues.** Prompts firing for / targeting the wrong player. Suspect: the card-selection
  `CardOptionPrompt` (steal builds options over the **target's** hand but `PlayerId` is the holder),
  `ContextPlayer`/`NearestPlayerAhead`/`DiceOffPlayer` resolution, and any-player (`MetAnyPlayerTurn`)
  prompts surfacing on the wrong drawer. Check `PromptPlayers`/`PromptForCard` `PlayerId` vs the hand
  being shown, and the dice-off prompts during a trigger.
- **Purging issues.** Purge actions misbehaving — `BuildingActionService` purge (self / chosen, count),
  `PurgingService.PurgeOwnProperty`/`PurgeOthersProperty`/`PurgeProperties`, and the `SwapSet`
  double-purge. Check which properties get purged, the count, and the prompt for the chosen-player purge.
