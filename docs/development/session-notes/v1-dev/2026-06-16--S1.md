The session pushed the **cards sub-engine** through the families that were deferred at the end of
2026-06-15: the **Dice / triple-bonus** family, the **Tax / trigger-amount** money seam, the
**jail-lock / collect-rent-in-jail** flags, and a generic **dice-off** picker shared across actions.

`Task<bool>` and takes an optional `CardActionContext`** — touching every action service and
`CardService`.

## 1. 🌟 Dice / triple-bonus family (was the big deferral)

New files:
- **`Enums/Cards/DiceKind.cs`** — `ModifyTripleBonus` / `ConvertDoubleToTriple` / `DowngradeTripleToDouble`.
- **`Models/Cards/Actions/DiceAction.cs`** — `Kind`, `Target`, and the payout params
  `PayoutFactor` (0 suppress / 2 double), `PayoutMultiplyByDie`, `PayoutRedirectToLowestRoller`.
- **`Models/Cards/DiceOff.cs`** — the generic dice-off config (`DiceCount`, `Highest`,
  `IncludeHolder`), reusable by *any* action that targets a dice-off winner.
- **`Services/Cards/Actions/DiceActionService.cs`** — resolves the payout family via the split
  `PlayerService.ApplyTripleBonus`; the conversions set `GameModel.ModifiedDiceRollType` for the
  orchestrator to re-route.
- Registered in DI (`MP.GameEngine/Extensions/ServiceRegistration.cs`), added to `CardService`
  switch + `[JsonDerivedType(typeof(DiceAction), "Dice")]` on `CardAction`.

**`PlayerService.ResolveTripleBonus` split into payout-vs-accumulator**: the old method becomes a
thin `=> ApplyTripleBonus(..., factor:1, recipient:null, ...)`. New `ApplyTripleBonus(holder,
factor, recipient, ct)` credits `base × factor` to the recipient (or holder), while the holder's
**accumulator always increments +£500** — even when the payout is suppressed (factor 0) or
redirected. Prompt body now branches on suppressed / redirected / normal.

**`GameModel.ModifiedDiceRollType` (`DiceRollType?`)** — new transient field set by the conversion
cards; cleared in `TurnStateProvider` (three reset sites). `DiceRoll` gained a
`DiceRoll(DiceRoll, DiceRollType)` copy-ctor (rebuild with a modified roll type) and
`IsDoubleFive` now keys off `RollType == Double` instead of `ThirdDie != 5`.

**`PlayerTurnOrchestrator` rewritten** (the bulk of the diff): the giant `switch` is decomposed into
`HandleNormalRoll` / `HandleDoubleRoll` / `TripleRoll` / `HandleTripleRoll`. A double that a card
upgrades to a triple credits the triple bonus and routes through the triple path; a triple
downgraded to a double routes through the double path. `EndPlayerTurn` reads
`ModifiedDiceRollType ?? diceRoll.RollType` to decide the extra roll. New rule codes
`Event_ConvertedToTriple` / `Event_DowngradedToDouble` (a converted-to-triple still clears global
events, a downgraded-to-double does not).

**`DiceService`** gained the shared dice-off seam: `RollDiceOff(candidates, diceCount, highest, ct)`
and `ResolveDiceOffTarget(holder, DiceOff, ct)` — the single picker behind money counterparties,
turns dice-off, and the triple-bonus redirect.

## 2. 🌟 Generic dice-off — replaced the Money-only `HighestRoller`/`LowestRoller`

`MoneyCounterparty.HighestRoller`/`LowestRoller` collapsed into a single **`DiceOffPlayer`**
configured by the action's `DiceOff`. `MoneyAction.IncludeHolderInRoll` removed; `MoneyAction.DiceOff`
(`DiceOff?`) added (also used as the swap partner when `SwapCash` has no chosen player).
`MoneyActionService`'s private `RollDiceOff` deleted — it now calls
`DiceService.ResolveDiceOffTarget`. **`TurnsAction` gained `DiceOff`** and `TurnsActionService` now
applies turns to a dice-off winner ("the lowest player misses 3 turns") — the dice-off is now a
generic non-money consequence picker.

## 3. 🌟 Tax / trigger-amount money seam

- **`Enums/Cards/AmountSource.cs`** — `Fixed` (default) / `TriggerAmount`.
- **`Models/Cards/Actions/CardActionContext.cs`** — carries `TriggerAmount` (the figure the firing
  trigger supplied), threaded as an **optional** param through `CardService.PlayCard → ResolveCard →
  ApplyAction →` every action handler. Only `MoneyActionService` reads it.
- `MoneyAction.AmountSource` added; `MoneyActionService.RealiseAmount` resolves `TriggerAmount` as
  `context.TriggerAmount × Amount` (Amount reused as the factor: ×3 "triple tax", ×1 "exactly that");
  no context → 0, a silent no-op.
- **Multi-target dice multiplier fix**: each subject now rolls its *own* dice multiplier (the prompt
  and figure belong to the affected player, not the holder).

This seam greens steal-FP, next-payment-doubled, GO-bonus, and the whole Tax deck at the **action**
level (the held trigger firing + `OnTaxLanded` trigger flag remain in the trigger layer).

## 4. 🌟 Jail lock / collect-rent-in-jail

- **`PlayerModel`**: `MinJailTurns` (`ushort?`), `CollectRentInJail` (bool), and
  `CanLeaveJail` (`[JsonIgnore]` computed — `!IsInJail || MinJailTurns == null || JailTurnCounter >=
  MinJailTurns`). Both new fields copied in the clone ctor.
- **`JailAction`**: `MinJailTurns`, `CollectRentInJail`. `JailActionService` only applies the
  term/lock/rent config when `SendPlayerToJail` actually jailed the player (guards against the
  JailFull event, where the fee is paid *instead* of jailing).
- **`JailService`**: `SendPlayerToJail` and `LeaveJailByCard` now return `bool`; every exit
  (`CheckAndLeaveJail`, `ForcePlayerToLeaveJail`, `LeaveJailByCard`) is gated by `CanLeaveJail` and
  cites `Jail_CantLeaveDueToCard` when blocked; lock/rent flags reset on exit. `PayJailFee` extracted
  as a private helper. Currency displays switched to `:N0`.
- **`PropertyService.PayRent`**: jailed owner with `CollectRentInJail` now collects (cites
  `Jail_CollectRentInJail`); switch over `owner` null / jailed+collect / jailed.
- **`TurnStateProvider.CanLeaveJail`** command gate now also checks the player's `CanLeaveJail`.
- **`MovementService`**: a jailed player can only be advanced to Just Visiting (the legitimate
  release); every other card advance no-ops and leaves them jailed. Emits the "released from jail"
  notification.

New rule codes: `Jail_CantLeaveDueToCard`, `Jail_CollectRentInJail`.

## 7. 🌟 Engine notification (toast) seam — end to end

- **`IEngineNotifier.Notify(gameId, targetPlayerId, message)`** — lightweight, non-pausing,
  fire-and-forget; distinct from a `Prompt`.
- **`SignalrEngineNotifier.Notify`** broadcasts a new `NotificationMessage(GameId, TargetPlayerId,
  Message)` record as a `"Notification"` event.
- **`wwwroot/scripts/InGame/game-notifications.js`** (new) — shows the toast *only* on the target
  player's surface (phone `[data-player]` keyed on `data-user-id`, or the host drawer when open on
  that player), mounted outside the StateChanged-swapped region, auto-dismiss 5s. Wired into
  `Player/Index.cshtml` and `Play.cshtml`.
- **`player-drawer.js`** publishes the open player via `drawer.dataset.openUserId` so the toast can
  target the drawer.
- Engine emits notifications from `JailService` (sent-to/released-from jail) and `MovementService`
  (released from jail).
