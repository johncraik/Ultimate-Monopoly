## 11. 🌟 Identity hardening

- **Redirect signed-in users off the auth flow** — `RedirectAuthenticatedFilter` (`IAsyncPageFilter`, mirrors
  `GithubManagerPageFilter`) on the Identity **`/Account`** folder → bounces authed users to **Manage/Index**.
  Skips `Manage/*`, **Logout, ConfirmEmail, ConfirmEmailChange, AccessDenied, Disabled** (still authed-usable);
  everything else (Login/Register/2FA/recovery/external/forgot-reset) redirects.
- **Password policy + email confirmation** — `Program.cs` `Configure<IdentityOptions>` (after `AddIdentity`):
  **≥8 chars, upper/lower/digit/special**, unique email; **`SignIn.RequireConfirmedAccount = true`** (the flag the
  scaffolded Register/ExternalLogin already branch on → no auto-login until confirmed). John also added lockout
  (5 / 30min) + allowed-username chars.
- **Register error display** — Identity/profanity rejections weren't showing: `asp-validation-summary="ModelOnly"`
  (and field spans) get **cleared by jQuery-unobtrusive for server-only errors** on interaction. Fixed by rendering
  **model-level errors in a plain `alert-danger`** (manual `@foreach` over `ModelState[string.Empty].Errors`), and
  **moving the profanity error from the `Input.Username` key to `string.Empty`** so it lands in that alert. Client
  password min-length synced 6→8.
- **Confirm-email pages restyled + backdoor removed** — `ConfirmEmail`, `ConfirmEmailChange`, `RegisterConfirmation`
  reworked into proper centred success/error cards (icons, app buttons; `Succeeded` flag drives state). Removed the
  scaffold's **`DisplayConfirmAccountLink` backdoor** (confirm-without-email link). John added an
  already-confirmed → login short-circuit; fixed its path `"/Login"` → **`"./Login"`** (absolute resolves to a
  non-existent area-root page).
