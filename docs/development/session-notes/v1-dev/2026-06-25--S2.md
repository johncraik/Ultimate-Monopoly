## 2. 🌟 Data Trail — two pages

- **Index** (`/Admin/Audit/Data/Index`) — a list of every audited table: **Table · Audits · Latest
  Audit · view button**. No search/pagination (the distinct-`TableName` set is small + bounded). Rows
  navigate via the shared `clickable-rows.js` (`data-href`); the eye button is the no-JS fallback. Bare
  `table-responsive` (no card), matching the other index pages. Backed by
  `GetAuditTableNames()` → `List<AuditDataTableViewModel>` (John's VM: TableName / AuditCount /
  LatestAuditDate, "Never" fallback).
- **Table** (`/Admin/Audit/Data/Table/{tableName}`) — one table's full change history. **Mirrors the
  User `Trail` page**: `{tableName}` is a **route param**, search + Action radios, AJAX over the shared
  `_AuditTable` partial via `audit-trail.js` (it keys off `location.pathname`, so the `{tableName}`
  segment is preserved on re-fetch). Page model named **`DataTableModel`** — *not* `TableModel`, which
  would collide with its own `public AuditTableModel TableModel` property (CS0542).
- **Sidebar:** Data Trail link wired from `#` → `/Audit/Data/Index`.

## 3. 🌟 `_AuditTable` made dual-mode (the reusable seam)

The same partial now serves both trails; its **second column flips** on a mode flag:
- **User trail** — the *user* is fixed → show the **Table** column.
- **Data trail** — the *table* is fixed → show the **User** column (username + truncated copyable id).

The flag is carried **two ways**: `AuditEntryViewModel.IsUserTrail` (per-row, John added it, set in the
service) **and** a new table-level **`AuditTableModel.IsUserTrail`** I added, mirroring the
`UserTableModel.FullTable` convention — the partial keys off the **table-level** flag so the header is
correct even on an **empty** page. `TrailModel` passes `true`; `DataTableModel` passes `false`.

## 4. 🌟 `AuditTrailService` — John's query layer (defects I flagged, he fixed)

- **`GetDataTableTrail`** built VMs with `isUserTrail: true` (copy-paste from `GetUserTrail`) → fixed to
  **`false`** so data-table rows show the User column.
- **`GetAuditTableNames`** was `.Select(...).GroupBy(key).ToListAsync()` — a terminal `GroupBy` returning
  groupings, which **EF can't translate** (runtime throw). Rewritten to aggregate in the projection
  (`GroupBy(TableName).Select(g => new { g.Key, g.Count(), g.Max(AuditDate) })`) → a real SQL `GROUP BY`;
  VMs + local-time formatting built in memory after.
- **Search semantics are now scope-dependent** (John's design, locked): one `Query` method branched on
  `string.IsNullOrEmpty(userId)` — scoped to a **user** (user trail) → search matches **TableName +
  EntityKey**; scoped to a **table** (data trail) → search matches **UserName / UserId + EntityKey**.

## 5. 🌟 Styling — admin table rows were too thin

The other index tables (`_UsersTable`, `_ReportsTable`) only *looked* taller because their cells carry
**two lines** (username + display name); the audit/data tables have single-line cells, so `table-sm`'s
`.25rem` vertical padding left them cramped. **Fix** (`admin.css`): a shared rule giving both admin table
families (`.admin-users-table` + `.admin-audit-table`) **`.6rem` top/bottom cell padding** — evens the
row rhythm across **every** admin list table. Specificity `(0,1,3)` beats Bootstrap's `.table-sm` rule
`(0,1,1)`, so no `!important`. The audit **detail** panel keeps its flush `p-0` (utility `!important`
wins) and empty-state cells keep `py-4`, so nothing else shifts. Also dropped the card wrapper from the
Data Index so its table sits bare like the sibling index pages.

Build green throughout (0 errors; warning count actually dropped vs the pre-session baseline).

## 6. Decisions locked this session

- **Data Trail = two pages**: Index (filterless table list) → Table/`{tableName}` (per-table history,
  mirrors User Trail). Index isn't searched/paginated — the table set is small + bounded.
- **`IsUserTrail` picks the variable column** — user trail fixes the user → Table column; data trail
  fixes the table → User column. Carried at the **table level** (`AuditTableModel`) for empty-page header
  correctness, alongside the per-entry flag on the VM (partial keys off the table-level one).
- **Search scope is mode-dependent** — user trail searches TableName + EntityKey; data trail searches
  UserName/UserId + EntityKey.
- **`GetAuditTableNames` aggregates server-side** — never a terminal `GroupBy` returning groupings.
- **Admin list tables share a `.6rem` row padding** — single-line audit/data rows match the two-line
  user rows; Data Index table is bare (no card), like the other index pages.
