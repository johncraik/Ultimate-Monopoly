## 6. 🌟 The privacy refactor — keep metadata **off the public repo**

The pivotal decision. Initially I folded **identity** + **client** metadata into the GitHub issue **body**
as HTML `<details>`/`<table>` blocks, and added the **`HtmlSanitizer`** NuGet (`9.0.892`, `Ganss.Xss`,
with `details`/`summary` allow-listed) to sanitise the user's free text + render with `@Html.Raw`. John
then (rightly) **killed all of that** — posting user/role/client info to a **public** repo is too risky
"no matter if it should be alright". The new shape (John's refactor):
- `BugReportService.RecordIssue(desc, type, userId, creatorName, **metadata**)` — metadata threaded
  **through JC.Github** and **stored on the `ReportedIssue` record** (`ClientMetadata`), **never in the
  description**. Sanitiser + `Html.Raw` removed (description is now plain text).
- `ReportedIssueViewModel` gained `ClientMetadata` (`Dictionary<string,object?>`, deserialised);
  identity is just the existing Reporter columns.
- **Identity exposure (locked):** username + email already excluded; we **dropped role / 2FA / restricted
  / hidden / enabled** too — advertising admins or un-MFA'd accounts on a public repo is recon/targeting
  risk. The accordion is now a **50/50 row**: full description (left) · **collapsible client-metadata
  table** (right, `-` for null) — then screenshot + comments full-width below.

## 7. 🌟 "View Issue in App" link — JC.Github `UpdateIssueBody`

Wanted a deep link in the issue back to `/Admin/Logs/Issues/Index#{ExternalId}` (and gave the table

## 8. 🌟 Synced badge + the Bootstrap **capture-phase** accordion bug

Made the GitHub column a green **"Synced - #N ↗" badge** where the **whole badge is the link**. Clicking
it kept toggling the row's accordion. Two failed bubble-phase `stopPropagation` attempts later, the root
cause: **Bootstrap 5's collapse data-API delegated handler is registered on `document` in the *capture*
phase** (it passes the delegation selector as the `useCapture` arg), so it fires *before* any bubble-phase
handler. **Fix:** a **`window` capture-phase** listener — the first thing to run for any click — that
`stopPropagation()`s for `.audit-row a`. CSP-safe (no inline `onclick`).

  `GithubManagerPageFilter` is now likely redundant / belt-and-braces.
