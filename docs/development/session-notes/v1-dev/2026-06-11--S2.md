## 2. 🌟 Three card bugs (against the TEMP deck)

### 2.1 Street-repair overflow → ~£4.29bn charge
A per-house/per-hotel charge with **no buildings** realises to `0`, then
`MoneyHelper.NormaliseAmount(0, …, CardCharge)` hits the **never-round-to-zero**
rule — and because `positive = amount > 0` is **false at 0**, it returns the
**negative** grid minimum (e.g. `-50`). Back in `RealiseAmount` that's
`(uint)(-50L)` → ~4.29bn, and `ApplyMoney`'s `if (amount == 0)` guard never fires.
**Fix:** short-circuit a genuinely-zero realised amount before normalising —
`if (amount <= 0) return 0;` (also catches a percentage that floors to nothing).
Landed in `MoneyActionService.RealiseAmount` after the §3 refactor.
> Flagged but left to John: the *root* footgun is `NormaliseAmount` returning a
> negative/`uint`-wrapping value from a literal `0` — the deeper fix is guarding
> the bump with `&& amount != 0` in the shared helper. Not done (shared surface).

### 2.2 Card fines must go to Free Parking
Every `MoneyDirection.Pay` action in `TEMP_CARDS` used `MoneyCounterparty.Bank`;
`game-rules.md` Default rule 5 — all fines go to Free Parking. Changed them all
to `FreeParking` (the `EachPlayer` player-to-player cards left alone — not fines).

### 2.3 🔴 Advance-to-GO doesn't release the initial-GO lock — and the trap
`HasPassedInitialGo` is set **only** in `CollectGoMoney` (the *passing* path).
An "Advance to GO" card lands on GO via `AdvanceIndex`, which returns
`passesGo:false` for a GO destination, so the flag is never set.
- **My first fix (`LandOnGo` sets the flag) — wrong, John caught it.** A **double
  3 at the start** wobbles `+3/-3` and lands back on GO via `MovePlayer →
  ResolveBoardSpace → LandOnGo`, so patching `LandOnGo` would wrongly release the
  lock on that wobble and let `FlipDirection` illegally flip them — the exact
  soft-lock Movement rule 2 guards against.
- **Correct fix:** the distinction is the *movement kind*, not the destination.
  `AdvancePlayer` (deliberate advance) sets `HasPassedInitialGo` when
  `newIndex == GoSpace`; `MovePlayer` (the wobble, normal rolls) is untouched, so
  genuine passes still set it via `CollectGoMoney` and a wobble-back-to-GO stays
  locked.

## 6. 🌟 `PlayCard` — the held-card lifecycle seam

apply the effect only** (where the future NOPE/immunity interrupt window will

unnecessary `JailService.LeaveJailByPlayingCard` wrapper (John: just inline
`PlayCard` in the pump).

## 7. 🌟 The "missing card" mystery — #1 = #3

- **#1 "Get Out of Jail Free not in the deck lists"** — they're the *only*
  keep-until-needed cards in TEMP; once drawn they're `Take`n out of the deck and
  held in `player.Cards` (every other TEMP card is resolve-on-draw and cycles
  back). So "missing from the deck" = sitting in a hand. Correct by design.
- **#3 "sometimes a player doesn't draw a Chance/ComChest"** — **same cause**:
  drawing a *keep* card landed it in the hand **silently** — only the
  resolve-on-draw path opened an acknowledge — so it looked like no draw.
  **John's fix:** hoist the acknowledge to the **top of `DrawCard`** (after the
  empty-deck guard) so every draw shows "you picked up X" once, covering both
  modes, and removed it from `ResolveCard`. Consequence (correct): a card *played
  from hand* via `PlayCard` no longer re-prompts the text — the player already saw
  it and is deliberately playing it (a multi-group card still opens its choice).
