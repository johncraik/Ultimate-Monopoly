## 2. 🌟 `cards-design.md` — designed end to end

John wrote a crude first pass; developed it into a full architecture doc over several turns
(the cards counterpart to `game-engine.md`). Key decisions (John's calls):

- **Cards are data, not a DSL.** A card = `Groups` (OR/choice) → `Actions` (AND) +
  `Conditions` + `CardType`. The closed **action vocabulary** is the `game-engine.md` §11
  "data-row-keyed-to-effect-enum" option — *not* the open interpreter previous attempts
  died on. Most actions thin-dispatch to existing services; **some add genuinely new code**
  (NOPE, in-jail rent, swap) — fine, because the *set* is closed.
- **Two interaction modes:** (a) active play of held cards; (b) **override-on-draw** —
  land → draw → the card's action supersedes the default (what every `// TODO take card`
  board stub awaits). A drawn card resolves immediately ⇒ `ConditionType.None`.
- **Conditions = `CardConditionType` (forced/choice × cardholder/any turn, + None) +
  `CardTrigger`** (a `[Flags]` event set; the trigger points ≈ the existing `CiteRule`
  branch map, so the held-card hook rides those rather than a new event bus).
- **Immunity is *reactive*, not a silent flag** (John's correction): a held card the
  *impacted* player is prompted to play to counter an action — same mechanism as NOPE
  (specific counter vs universal). Both ride one response window (the to-be-realigned
  `InterruptibleWindowPrompt`). NOPE built last.
- **Global-event cards** are all Double cards, and a double clears the current event ⇒ the
  single-slot `GlobalEventInfo` can never hold two. **Percentage cards** read the existing
  `PlayerPercentCap`.
- **Decks are queues:** shuffle once into a per-type `Queue` on `GameModel`; draw = front,
  return = back; the order *is* the deck and snapshots/replays deterministically.
- **Import last** — settle the live model + resolution first, then build the import object.

## 6. 🌟 `CardService` — the interpreter

New `Services/Cards/CardService.cs` (cards get their own folder tree — the mini-engine). On
the **`GameEngine` bundle** (`engine.CardService`, like `ShortfallService`) so `BoardService`
reaches it without a ctor dep — breaking the otherwise-circular `BoardService ↔ CardService`
(card movement resolves a landed space, which can draw a card…). DI-registered (John).

- **`DrawCard(type)`** — `Take` from the deck; `!IsKeepUntilNeeded` ⇒ `ResolveCard` +
  `HandBack`; else add to hand + `CardTakenReceipt`. No-op on empty deck.
- **`ResolveCard`** — single group applies directly; >1 ⇒ `CardOptionPrompt` keyed by the
  stable `GroupId`; applies the chosen group's actions in order; emits `CardPlayedReceipt`
  (a resolve-on-draw card still counts as "played").
- **`ApplyAction`** — one switch dispatching to the three handlers.
- **Handlers** (I built; John refined): **Money** → `RealiseAmount` (base × per-unit count ×
  %cap, then `MoneyHelper.NormaliseAmount` with `CardPayout`/`CardCharge`) → `TransactionService`
  for Bank/FreeParking. **Movement** → target-resolve → `MoveSpaces`/`AdvanceToIndex`/
  `AdvanceToNearest`/`GoToJustVisiting` via `MovementService` + `BoardService.ResolveBoardSpaceForPlayer`
  (gated by `ResolveLandedSpace`); **swap** is its own path (direct index exchange +
  `PlayerSwappedReceipt`, no GO, no resolve). **Jail** → `SendToJail` (+ `TurnsOverride` →
  `MaxJailTurnsOverride`) / `Release` (`LeaveJailByCard` when in jail). Shared
  `ResolveTargets` (Self / AllOthers / `ChosenPlayer` via `TargetPlayerPrompt`),
  `FindNearest` (scans forward in the player's direction).
- **`%cap` bug (John's refactor, I caught):** `amount *= (cap / 100)` integer-divides
  `50/100`→`0` and `10/100`→`0`, zeroing every non-double-hotel percentage card. Fixed to
  multiply-before-divide.

## 7. Card-ID persistence — reviewed

`PersistedCardIds` (keyed by unique `CardText`, stores the card/group/action/condition GUID
tree as JSON) + `CardImportService.GetAndPersistCardIdObj` (find-or-create, stamp the GUIDs
onto the `CardModel`) + the `AppDbContext` `DbSet`. The point: cards live in snapshots **by
id**, so a re-import must hand back the *same* GUIDs. Review landings (all John's calls):
matching is **positional on identical file order** (the `OrderBy(GroupText)` was the odd one
out vs the positional actions/conditions — **dropped** it for uniformity); editing a card
**requires manually deleting its row** (intentional — forces drift to a new id, protects
integrity); not soft-deleted (so the unique-index clash is moot); the import object →
`CardModel` translation is the remaining TODO (import is last).

## 11. Street-rule fix (review + John's fixes)

The half-price-on-a-full-street mechanic (`StreetRuleQualifier`). Caught two real defects:
- 🔴 **`NormaliseProperties` crashed on any owned station/utility** — `StreetPartner[set]`
  (the throwing indexer) has no Station/Utility key, and they reach that line. Newly
  introduced by the fix. John guarded `if (set is Utility or Station) continue;`.
- 🟠 **`StreetRuleQualifier.BuiltOn` was never assigned** — the build path didn't record it,
  so a set built-on-then-completed wrongly qualified for the discount. John set it in
  `BuildOnProperties` (preserving `Qualified` so an already-qualified street keeps the
  discount while building).
- **The lose-and-regain edge:** the reset block demotes a qualified-built-on property to
  `BuiltOn` when the street breaks. My suggested `&&`→`||` (fire on *either* set incomplete)
  was right but **incomplete — without a `continue`, the demoted property falls through to
  the re-qualify block in the same pass**. John added both (`||` + `continue`); I'd dropped
  the `continue` from my recommendation (my miss).
