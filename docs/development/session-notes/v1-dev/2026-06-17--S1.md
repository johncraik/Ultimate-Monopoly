## 9. ­čö┤ OPEN BUG ÔÇö `SuppressDefault` aggregation aliases & mutates a card definition

**Where:** `CardTriggerService.EvaluatePlayableCards` (the suppress aggregation), interacting with
`CardService.PlayCard`'s return and `CardModel`'s copy ctor.

**The chain:**
1. `PlayCard` returns **`card.SuppressDefault`** ÔÇö the card's *own definition object* ÔÇö when a card
   applies (only the not-applied path returns a fresh `new SuppressDefault(None)`).
2. `CardModel`'s copy ctor **shares the `SuppressDefault` reference** across every per-turn working
   copy and the persisted deck definition (treated as immutable card data ÔÇö ┬ž1).
3. `SuppressDefault.Aggregate(other)` **mutates `this`** (ORs `other`'s flags in).
4. The loop does:
   ```csharp
   if (completeSuppress != null) completeSuppress.Aggregate(suppressDefaults);
   else                          completeSuppress = suppressDefaults;   // ÔćÉ aliases card #1's definition
   ```

**Effect:** when **Ôëą2 cards fire on one trigger**, the first card becomes the accumulator and
`Aggregate` **permanently ORs every later card's suppress flags into card #1's definition**. Because
the definition is shared by reference, card #1 then suppresses things it never should ÔÇö for the rest
of the game, on every future draw/play. With exactly one card firing (common case) it's currently
harmless (the alias is only read), but fragile.

**Reachability:** John confirmed 1ÔÇô2 cards can fire on the same trigger, so the 2-card case is real.

**Fix (one change):** never let a card's `SuppressDefault` *be* the accumulator ÔÇö start a fresh one
and always aggregate into it:
```csharp
completeSuppress ??= new SuppressDefault(SuppressDefaultType.None);
completeSuppress.Aggregate(suppressDefaults);
```
(`SuppressDefaultType` is already in scope.) Leaves all card definitions untouched; the existing
`return completeSuppress ?? new SuppressDefault(None)` paths still hold.

**Underlying footgun (the real lesson):** snapshot copy ctors **share** the immutable card-definition
references (`Groups`, `Conditions`, `SuppressDefault`). Anything that *mutates* one of those objects
in place corrupts the shared definition across the whole game. Aggregation/derivation must always
build a **fresh** object, never mutate a returned definition. (This is the third copy-ctor/shared-ref
incident this stretch: `GlobalEventInfo`, `SuppressDefault`-dropped, now `SuppressDefault`-mutated.)
