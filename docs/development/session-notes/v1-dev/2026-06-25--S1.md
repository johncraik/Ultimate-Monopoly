## 4. рџЊџрџЊџ THE BIG ONE вЂ” snapshot card dedup (~200KB в†’ ~12KB per turn)

The size columns revealed snapshots were **~200KB/turn**, ~95% of it the **card decks**: `GameModel.CardDecks`
held full `CardModel`s (text + groups в†’ actions в†’ params + conditions + GUIDs) for all ~170 cards, re-serialised
every turn. Events were already lean (0вЂ“2KB). Two refactors (John built, I reviewed) deduped it:

### 4a. Deck refactor вЂ” decks store card ids
- `CardListModel` decks в†’ `Queue<string>` (CardIds). `Take(ICardCacheService cache, CardType)` dequeues an id
  and resolves the `CardModel` from the cache. New engine abstraction **`ICardCacheService`**
  (`MP.GameEngine.Abstractions.Cards`), implemented by the web `CardCacheService`; DI-wired
  (`TryAddScoped<ICardCacheService, CardCacheService>`); warmed in `Program.cs`. `CardDeckHelper.BuildCardDecks`
  now populates id queues; the copy-ctor copies string queues (cheap).
- **рџ”ґ Critical bug I caught:** `Take` returned the **shared cached `CardModel`** (`cache.GetCard(id)`), which
  `CardService` then mutated (`IsChosenGroup`, `DrawnOnTurn`) and added to hands вЂ” corrupting the process-wide
  singleton across games. Fix: **`Take` clones вЂ” `new CardModel(await cache.GetCard(id))`** вЂ” the single
  choke-point so a caller can never leak the shared instance. (John applied it; I fixed the XML docs.)

### 4b. Player-hand refactor вЂ” thin `PlayerCardInstance`
- `PlayerModel.Cards` (List<CardModel>) в†’ **`CardInstances`** (List<`PlayerCardInstance`>). A
  `PlayerCardInstance` carries only the mutable per-instance state: **`CardId`, `ChosenGroupId`,
  `TurnsRemaining`, `DrawnOnTurn`**. `new CardModel(model, instance)` clones the cached def + overlays the
  instance. `PlayerModel.GetCards(cache)` + the async `GetOutOfJailCard(s)` / `GetPlayableCards` /
  `GetOtherCards` helpers reconstruct on demand. Consumers migrated: `CardService`, `CardTriggerService`,
  `BankruptcyService`, `JailService`, `PlayerProfileService`, `CardTransferActionService` (moves the thin
  instance), `CardImmunityService`, `_PlayerProfileView` (injects `ICardCacheService`, awaits the accessors).
- **рџ”ґ Bug I caught:** `DrawnOnTurn` was dropped (instance didn't carry it) в†’ the `OnNextMove` gate in
  `CardTriggerService` was permanently bypassed (the JustVisiting +23/в€’17 cards would fire on the move that
  drew them). John fixed it (added `DrawnOnTurn` to the instance + `CardModel(model,instance)` overlay).
- **рџџЎ Minor I caught:** the `!applied` (blocked jail-release) path recorded a spurious chosen group. John
  fixed `UpdateInstance` to `ChosenGroupId = chosenGroup.IsChosenGroup ? GroupId : null`.

### 4c. Result
Snapshots **~200KB в†’ 11-13KB** (~16Г—), events still 1вЂ“4KB. A **16-turn game = ~230KB total** вЂ” roughly what a
*single* turn used to cost. Pure dedup, no fidelity lost (full defs reconstructed from the cache). Matches the
design doc's original "a few KBвЂ“tens of KB" estimate.

## 5. рџЊџ JC.Core `EntityKey` migration saga (MySQL)

John added the `AuditEntry.EntityKey` column to JC.Core (unblocks the Audit data trail). The migration fight:
- `varchar(1024)` в†’ MySQL **3072-byte index limit** blown: `(TableName 256 + EntityKey 1024) Г— 4 (utf8mb4) =
  5120`. Dropped to **512** в†’ `(256+512)Г—4 = 3072` exactly (the inclusive max вЂ” fits, zero margin).
- Then a **`Duplicate column name 'EntityKey'`** on re-apply. Root cause: **MySQL DDL isn't transactional** вЂ”
  the first attempt's `ADD COLUMN` committed but the `CREATE INDEX` failed, so EF never recorded the migration
  in `__EFMigrationsHistory`. The orphan column survived (its `Down` never ran). Fix: **drop the orphan column
  by hand** (`ALTER TABLE AuditEntries DROP COLUMN EntityKey;`) then re-run `database update`. John cleared it.

## 11. Decisions locked this session

- **Snapshot card dedup:** decks = `Queue<string>` (ids); hands = `PlayerCardInstance` (CardId / ChosenGroupId
  / TurnsRemaining / DrawnOnTurn); full defs come from `ICardCacheService` on demand; **`Take` always clones**
  so game state never aliases the shared cache.
- **Breaking snapshot format:** old snapshots (full `Queue<CardModel>` / `List<CardModel>` shape) won't
  deserialise into the new id/instance shape вЂ” accepted (pre-V1; card edits already need a fresh game).
- **`EntityKey` is a JSON object** (corrected В§9.1); shipped `varchar(512)` so `(TableName,EntityKey)` stays
  в‰¤ 3072 bytes.
- **Revert semantics:** revert to turn N = resume at the **start of N** (re-play it) в†’ delete later turns +
  the **target turn's events row**, keep its GameTurn + GameSnapshot, tear down the live runtime.
- **`AdminActionType` is `int`-persisted** в†’ new values append at the end (`GameReverted`).
- **Recent Activity panel** = reusable atoms (scoped query + table partial + preview/full); **Recent games is
  SystemAdmin-only**, the rest are any admin tier; admin-logs = `CreatedById` + the former-admin visibility
  rule.
- **Audit display:** table + **independent** accordion rows; the badge colour map; **amber** (not red) flags;
  pretty `<pre>` JSON; **userId as a route param**; folder `Audit` (singular).
