## 1. 🌟 R — the 8 live-play bugs (R-01–R-07 fixed, R-08 deferred)

John brought 8 bugs hit in real games; appended to `review.md` as `R-01..R-08`, then worked them.

- **R-01 — prompt requiring more selections than options locks the game.** `TargetProperty/PlayerPrompt`
  enforced *exactly* `Count` at both the client gate and `PromptValidator`; "purge 2" with 1 eligible
  property was unsatisfiable. Fixed in **4 layers**: prompt-model backstop (`Count` clamps to options),
  validator (`Math.Min`), client (`Math.min(count, available)`), and `PurgingService` call-site clamp +
  reworded title/body. Used `Math.Min` not the `|| options.Count` form (which would let you over-pick).
- **R-02 — cards don't play in jail.** A blanket `where !player.IsInJail || trigger==OnInJail` in
  `CardTriggerService.MatchingCardForTrigger` killed the dodgy-judge `OnRollDouble`+`JailFilter=OnlyJailed`
  card. **Decision (John): make ALL cards playable in jail** — deleted the filter entirely (reverses the
  18th-S2 "exclude jailed holders" decision). Confirmed GOOJF cards carry `Trigger:0` so they never
  trigger-surface (play via the `LeaveJailCard` command), and the `CanPortfolioCommand` `!IsJailed` gate
  means no movement-card jail-escape.
- **R-03 — re-purge the same property.** Removed the `!HasBeenPurged` eligibility filter; `BuiltOnProperties`
  already requires current buildings, so a rebuilt property re-qualifies. **John then removed the
  `HasBeenPurged` field entirely** (engine field + copy-ctor + `OwnProperty` reset + the profile "Purged
  Before" badge). `IsPurged` is the sole purge-state flag now.
- **R-04 — `NormaliseProperties` not run across card resolutions.** Injected `PropertyService` into
  `CardService` and call `NormaliseProperties` once at the `ResolveCard` chokepoint (covers every
  property-moving / multi-action card). No DI cycle.
- **R-05 — 0-building swapped set marked purged.** Guarded `PurgingService.PurgeProperties` to skip
  non-`BuiltOn()` properties (the `SwapSet` direct-purge passed the whole set).
- **R-06 — free-hotel tax card did nothing.** It only fired if you already owned a 4-house street.
  **John's spec, implemented:** new `PlayerModel.FreeHotels` credit; `GrantHotel` places a hotel now if
  pool+4-house street, else `FreeHotels++`; build consumption waives a hotel build's cost per credit
  (in the core `BuildOnProperties` loop + a `FreeHotelDiscount` preview in the 3 entry points so a
  cash-poor holder isn't blocked); "Free hotels" row on the profile property tab. John added a
  `Math.Min((ushort)2, diceCount)` hardening elsewhere.
- **R-07 — "each player hands in a property" FP card let the holder hand a just-received property back.**
  `ReceiveAllFreeParking` transferred to the holder before `ProcessFreeParking` computed hand-in
  eligibility. **Decision (John): option B, data-only** — removed `ReceiveAllFreeParking` from the card
  JSON; the **default FP take** now hands the properties over *after* eligibility is fixed. Trade-off
  accepted: the default take is money-gated (path A), so an empty-cash pot means no receipt — documented
  in the card text "**(if free parking has money)**" and in `cards.md`. (Needs a fresh game.)
- **R-08 — orchestrator/dice quirks. ⏸️ DEFERRED.** Vibes-based, not reproduced; read-through found the
  sequencing sound. Needs a unit/simulation test pass (roll-resolution paths + card bonus accumulator/
  payout) before any change. The in-jail converted-roll path from R-02 lands here too.

## 2. 🌟 H — high/critical

- **H-01 — completion clears live runtime before DB commit.** `GameCompletionService.ConcludeGame` tore
  down cache+pump *before* the conclude transaction → a rolled-back commit left a zombie. Moved
  `ClearGameRuntime` to **after** the commit (now conditional on durable success; pump stop stays
  fire-and-forget as it runs on that pump).
- **H-02 — board-skin cache invalidation blocked for normal users.** `BoardCacheService.Invalidate`
  returned early for any non-empty `userId` unless admin. **John reworked it:** `Invalidate(userId=null,
  bypassAdminCheck=false)` — own-cache is the no-arg form, another user's needs admin, and a
  `bypassAdminCheck` overload covers the share-recipient loop. Callers aligned (John did the 6
  `BoardSkinService` calls + the share loop; I did the one `TryRemoveSharedBoardSkin` own-cache miss).
- **H-03 — movement stats throw on no-landings.** `landOnIndexes.MaxBy(...)` throws on an empty
  `Dictionary<,>` (value-type element) → sank a whole game's stats. Guarded with sentinel `GoSpace`
  (field is non-nullable `ushort`; its land-count is already 0 → internally consistent).
- **H-04 — card import silently skips missing decks. ❌ BY DESIGN** (John): missing card JSON is
  tolerated, game plays fine without cards.

## 3. 🌟 M — medium

- **M-01 — `CardOptionPrompt` accepts any key.** Validator now requires empty-decline OR a real option
  key for `PlayCardChoice`; `CardImmunityService` plays only when `SelectedKey == immunityCard.CardId`.
- **M-02 — game-cache hydration not single-flight.** Added a `static ConcurrentDictionary<string,
  SemaphoreSlim>` per-game gate with double-checked cache lookup in `GetGame`. **John added** gate removal
  on `Invalidate` (undisposed — `WaitAsync`/`Release`-only semaphore allocates no handle, and disposing a
  held one would throw on `Release`).
- **M-03 — deals: docs say any-player-at-boundary, code required current player.** **Decision (John): any
  ACTIVE player may deal at a turn boundary** (the hub contract was right). Removed `IsCurrentPlayer` from
  `CanDeal`, added `IsActivePlayer` (`GetPlayer != null`); John extended the active-guard to `CanStartTurn`
  + `CanDeclareBankruptcy`.
- **M-04 — `GameExecutor` queue unbounded.** Bounded the pump channel to **10** pending. Added a tri-state
  `EnqueueOutcome` (`Enqueued`/`Full`/`Closed`) + `_closed` flag so the retry loop distinguishes a full
  queue (drop/backpressure) from a closing pump (recreate) — `TryWrite` returns false for both.
- **M-05 — stats write-once. ❌ BY DESIGN (for now)** (John): recompute = manual record delete + Hangfire
  re-fire; admin "refresh" may come later.
- **M-06 — jail card-exits miscounted as dice.** Added `IsJailRelease` to `CardPlayedReceipt` (chosen group
  has a `JailAction{Kind=Release}`); `JailStatsService` counts it as `leftByCard`. (Serialization-timing
  caveat — old games read 0; same shape as the immunity caveat.)
- **M-07 — card dice rolls not counted.** **Decision (John): card rolls emit a `DiceRollReceipt`** (safe —
  never a 3-dice turn roll). `cardRolls` = `!IsTurnRoll`; the dice-number stat gated to `IsTurnRoll &&
  RollType!=Triple`. John added a `diceCount` clamp + made `PlayerModel.IsDiceNumber` exclude `Triple`
  (engine + stat now agree).
- **M-08 — order-dependent card IDs. ❌ BY DESIGN** (John): index identity is fine; JSON edits need a fresh
  game anyway.
- **M-09 — import mismatch exceptions lack context. ✅ STALE/already-fixed** — current `TryPopulate` returns
  `false` on a count mismatch rather than throwing.
- **M-10 — `GetCard` false nullable contract. ✅ (John)** — changed to non-nullable `Task<CardModel>`
  (required getter that throws).

## 4. 🌟 L — low/polish (two were bigger than the review framed)

- **L-01 — rate-limit comment vs value.** Turned out the config leaned on **misleading TokenBucket
  defaults**: `PermitLimit` is a no-op under TokenBucket; `TokenLimit` (0→fell back to `PermitLimit` 20)
  was the bucket, `TokensPerPeriod` (default 10) the refill → actual behaviour was *burst 20 / sustained
  10/min*. After a token-bucket explainer + intent discussion (auth endpoints must tolerate legitimate
  fumbling-session bursts; per-account lockout is the real brute-force defence), set it **explicitly**:
  `TokenLimit=30` (burst), `TokensPerPeriod=15` (sustained), `Window=1min`, removed the no-op `PermitLimit`,
  accurate comment.
- **L-02 — stale host-panel TODOs. ✅ already cleaned** (John).
- **L-03 — setup SVG QR path. ✅ removed** (John) — redundant, lost to the post-create redirect.
- **L-04 — stats always use default board. PREMISE WRONG → fixed properly.** **John corrected me: board
  skins only RENAME spaces (not prices/colours/layout)** — so `game-engine.md` was wrong (I corrected it),
  and the `StatisticsJob` computation against the default board is numerically fine. The real fix is
  display: **John implemented per-page board selection across all 5 stat pages** — `Games/GameStats` +
  `Games/Compare` use the game's board (`BoardId`, fallback default); `Profile/Index`, `Friends/Profile`,

- **L-05 — `ShortfallAmount`/`AmountOwed` unsigned underflow.** Clamped by comparing before subtracting
